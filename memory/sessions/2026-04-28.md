### Decisions Made

1. **Title-length cutoff is 4 chars (`MIN_TITLE_CHARS=4`).** Spec called out 'CEO' as the documented footgun; 4-char minimum catches CEO/CFO/COO/CTO/CXO uniformly. 4-char titles like 'CMOs' / 'VP HR' pass the boundary. Whitespace-only titles (e.g. 4 spaces) caught via `len(strip()) < 4`.
2. **Ambiguous-geography set is exactly {'US', 'UK', 'AU'}.** Spec listed those three. Frozen set so the membership check is O(1) + future additions are explicit.
3. **All 4 tier-monotonicity violations reported simultaneously.** When operator inverts the entire ladder (A=10 < B=20 < C=30 < D=40 < archive=50), all 4 pair errors land in the same exception message — saves a multi-edit cycle.
4. **Validator does NOT enforce field presence.** Missing icp / tier_thresholds entries are permitted. The validator catches *invalid values*; field defaulting belongs to the use-site (e.g. score_stage falls back to DEFAULT_TIER_THRESHOLDS).
5. **Provision script does NOT run migrations.** Operator's responsibility — documented in the script docstring. The script assumes migrations already applied to the target Supabase project. Running migrations would require an admin connection + couples the bootstrap to migration state.
6. **`autonomy_rules` defaulted to `level='suggest'` for `send_email`.** Per CLAUDE.md "Start at 'suggest' for everything." Operator promotes after 50+ decisions / 80%+ success / 30+ days at level (per the autonomy progression rules).
7. **Idempotency split: DB vs folders.** Existing `clients` row → skip DB inserts entirely (single check). Existing folders → per-folder `already_present` accounting (some may exist, some may not — partial bootstrap is allowed). Both paths report cleanly without raising.
8. **`--dry-run` flag plans without writing.** ProvisionResult carries `would_create_paths` so the operator can preview. DB inserts also gated; existing-client detection still runs (read-only).
